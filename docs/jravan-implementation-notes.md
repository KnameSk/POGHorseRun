# JRA-VAN実Adapter 実装メモ

## 現在の状態

`IJraVanClient`、モック実装、実装用スタブは分離済みです。SDKなしでもアプリ全体のビルド・テスト・Discord通知フローを確認できます。SDK仕様が手元にないため、COM型名、データ種別、フィールド位置、コード体系は実装に固定していません。

## 仕様書で確認する箇所

1. JV-Link初期化・終了
   - COM生成、初期化メソッド、サービスキーの渡し方
   - 32/64bit要件、スレッドモデル、終了処理
2. データ取得
   - 出馬表または競走馬出走情報の正式なデータ種別
   - 期間指定の形式、差分取得の扱い、読み取り終了条件
   - エラーコード、リトライ可否、メンテナンス時間帯
3. レコードマッピング
   - 開催年月日、場コード、回次、日次、レース番号
   - 馬ID・馬名、騎手ID・騎手名、枠番・馬番
   - 芝・ダート・障害、距離、発走時刻、競走条件・クラス
4. コード表
   - 競馬場、トラック、競走条件、取消・除外状態の正式なコード
5. JRA公式URL
   - 2026-06-22時点で、JRA公式の個別出馬表は `JRADB/accessD.html?CNAME=pw01dde.../...` 形式です。
   - `CNAME` の全要素や末尾識別子を推測生成しません。公式仕様またはJV-Linkから安全に導出できる値が揃った場合だけ、`RaceUrl:Template` と `{raceId}` を使用します。
   - 揃わない場合は公式の出馬表開催選択 `https://www.jra.go.jp/JRADB/accessD.html` へフォールバックします。

## Adapter実装の方針

- JV-Link固有型は `Infrastructure/JraVan` の内部だけに閉じ込めます。
- 取得レコードを `RaceEntry` へ変換してからCoreへ返します。
- 生レコードは調査用に `RawData` へ保存できますが、秘密情報を含めず、ログ出力を制限します。
- COM呼び出しはキャンセル、タイムアウト、解放を明示し、同時実行を避けます。
- 仕様書で確認できない値は空値として扱い、推測コードへ置き換えません。
- 実Adapterの契約テストはSDK環境専用に分け、通常CIはモックだけを使用します。
